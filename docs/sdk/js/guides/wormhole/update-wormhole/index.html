Updating Your Wormhole App

When using a new version of the MoSync SDK or MoSync Reload, you should update your existing HTML5 apps. This guide takes you through the process.

What Needs To Be Updated?

To make sure that your existing apps work with the latest versions of the MoSync tools, note the following:

  • For MoSync Reload you have to update the file wormhole.js and make sure you are using the most recent Reload Client. Download it from the app stores, or go to the Reload download page, to get the latest version.
  • For apps you build using MoSync SDK you should update wormhole.js and check the C++ code in main.cpp.

Update wormhole.js

The file wormhole.js contains the JavaScript source code for the Wormhole library. Make sure your project uses the latest version of this file. This file is located in the LocalFiles/js folder in your project. Just delete the old version and paste in the new version.

If you use MoSync Reload, here are two alternative ways to obtain the new version of wormhole.js:

  • Create a new project with the new version of Reload, select the project and click Open. Copy the js/wormhole.js file and paste it into your existing project to replace the old version.
  • Get it from the Reload install directory. On OS X this is:
    /Applications/Reload/Contents/Resources/templates/ReloadTemplates/LocalFiles/js/wormhole.js
    On Windows and Linux the directory is:
    <directory-where-you-unpacked-reload>/server/templates/ReloadTemplate/LocalFiles/js/wormhole.js

If you use MoSync SDK, here two alternative ways to find wormhole.js (note that the wormhole.js used in Reload and the SDK is the exact same file):

  • Create a new project in the MoSync IDE using the HTML5/JS WebUI Template. Copy the LocalFiles/js/wormhole.js file and paste it into your existing project to replace the old version.
  • Get it from the MoSync SDK install directory. On OS X this is:
    /Applications/MoSync/templates/HTML5 JS WebUI Project/wormhole.js
    On Windows the directory is:
    <directory-where-you-installed-mosync>/templates/HTML5 JS WebUI Project/wormhole.js

Update your C++ code

Note that class WebAppMoblet has been deprecated, it is replaced by class HybridMoblet.

If you have an old project where main.cpp is not modified, you can just replace that one with the main.cpp from a project created with the most recent MoSync SDK.

If you have modified the main.cpp file you have to replace references to WebAppMoblet with HybridMoblet, and then manually update your code. The best way of knowing what needs to be changed, is to create a new HTML5/JS project in the MoSync SDK IDE (Eclipse) and inspect that code to see what changes have been made.
