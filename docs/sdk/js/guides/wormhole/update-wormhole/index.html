Updating Your Wormhole App

When updating your existing MoSync HTML5 apps that are built using Wormhole, there are some important steps to follow. This guide takes you through the process.

What Needs To Be Updated?

If you have existing apps that were created using MoSync SDK or Reload, you need to make sure they are updated to work with the latest versions of the MoSync tools:

  • For MoSync Reload you have to update the file wormhole.js and the Reload Client.
  • For apps you build using MoSync SDK you have to update to MoSync SDK 3.3, and update wormhole.js and the C++ code in main.cpp.

Update wormhole.js

The file wormhole.js contains the JavaScript source code for the Wormhole library. Make sure your project uses the latest version of this file. This file is located in the LocalFiles/js folder in your project. Just delete the old version and paste in the new version.

If you use MoSync Reload (version 1.1), here are ways to obtain the new version of wormhole.js:

  • Make a copy of this file from a project created with the new version of Reload, and paste it into your existing project to replace the old version.
  • Get it from the Reload install directory. On OS X this is:
    /Applications/Reload/Contents/Resources/templates/ReloadTemplates/LocalFiles/js/wormhole.js
    On Windows and Linux the directory is:
    <directory-where-you-unpacked-reload>/server/templates/ReloadTemplate/LocalFiles/js/wormhole.js

If you use MoSync SDK (version 3.3), here is where to find wormhole.js (note that Reload and the SDK uses the same version or wormhole.js):

  • Make a copy of this file from a project created with the HTML5/JS WebUI Template, and paste it into your existing project to replace the old version.
  • Get it from the MoSync SDK install directory. On OS X this is:
    /Applications/MoSync/templates/HTML5 JS WebUI Project/wormhole.js
    On Windows the directory is:
    <directory-where-you-installed-mosync>/templates/HTML5 JS WebUI Project/wormhole.js

Update the MoSync SDK

If you use MoSync SDK to develop Wormhole apps, or build native apps from your Reload projects, you need to install the MoSync SDK 3.3. Get it from the download page.

Update your C++ code

If you have existing projects you build using MoSync SDK, you need to update them if they were created with MoSync SDK 3.1.1 or earlier versions. Class WebAppMoblet has been deprecated, and is replaced by class HybridMoblet.

If you just created an HTML5 project created from a template without modifying any of the C++ code, you can create a new project using Reload 1.1 or MoSync SDK 3.3 and paste your HTML/JS files and resources into the new project. This should work out-of-the-box.

If you have projects with custom C++ code you have to replace references to WebAppMoblet with HybridMoblet, and then manually update your code. The best way of knowing what needs to be changed, is to create a new HTML5/JS project in the MoSync SDK IDE (Eclipse) and inspect that code to see what changes have been made.

Update the Reload Client

If you use MoSync Reload, make sure you have the latest version of the Reload Client installed on your devices and emulators/simulators. The old client will not work with the latest version of Reload. Go to the Reload download page to get the latest version.
