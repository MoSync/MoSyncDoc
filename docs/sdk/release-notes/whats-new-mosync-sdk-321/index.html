What's New in MoSync SDK 3.2.1

MoSync SDK release 3.2.1 is an update of release 3.2 that features updates to the Wormhole JavaScript library, along with several bug fixes and improvements.

Update your existing HTML5/JS projects: If you use the Wormhole library and have HTML5/JS projects created with previous versions of MoSync SDK, make sure to update the file wormhole.js to the latest version and to update the supporting C++ code in main.cpp.

Overview

The MoSync SDK 3.2.1 release is an incremental update of the previous MoSync SDK 3.2 release. Kindly consult the Release Notes for MoSync SDK 3.2 for a complete list of the many new features and updates that were introduced in this release. This document list the updates made for MoSync SDK 3.2.1.

Updating Existing Wormhole Projects

If you have HTML5/JavaScript projects that use the Wormhole library, you need to update the file wormhole.js and in some cases also the file main.cpp. For detailed instructions, please read the Wormhole Upgrade Guide.

Updating wormhole.js

For your existing HTML5/JS projects to work in the MoSync SDK 3.2.1, you must update the file wormhole.js to the most recent version. You can make a copy of this file from a project created with the new version of the SDK, and paste it into your existing project to replace the old version.

Updating your C++ code

You need to update the code in file main.cpp if you have HTML5/JS projects created with MoSync SDK 3.1.1 or earlier versions. Class WebAppMoblet (deprecated) has been replaced by class HybridMoblet. If you just created an HTML5 project from a template without modifying any C++ code, you can create a new project using MoSync SKD 3.2.1 and paste your HTML/JS files and resources into the new project. That should work out-of-the-box.

If you have modified the code in main.cpp, you need to manually update the code. For further details, see the Wormhole Upgrade Guide.

New Functionality in the Wormhole Library

The MoSync JavaScript library has been updated with new functionality. Updating Wormhole apps is essential if you have existing apps developed with previous MoSync SDK versions.

Screen Orientation Support in JavaScript

There is a new function for setting the screen orientation of the device called mosync.app.screenSetOrientation(). This function allows you to set the screen orientation to portrait, landscape, or dynamic mode. Examples:

mosync.app.setScreenOrientation(mosync.SCREEN_ORIENTATION_DYNAMIC);
mosync.app.setScreenOrientation(mosync.SCREEN_ORIENTATION_PORTRAIT);
mosync.app.setScreenOrientation(mosync.SCREEN_ORIENTATION_LANDSCAPE);

Open External Web Browser from JavaScript

If you want to open an external website from your app, you can open a URL in the web browser of the device using the new function mosync.app.openExternalURL(). Opening the URL directly in a WebView widget is also possible, but may not provide the support users need, like web navigation. You can also use this function to dial a telephone number directly from your app. Examples:

mosync.app.openExternalURL("http://maps.google.com/");
mosync.app.openExternalURL("tel:046850510300");

Improved Power Usage by Hybrid Moblets

We have improved the power usage of the HybridMoblet in the Wormhole C++ Library, dramatically reducing battery drain when an app is sent to the background. To take advantage of this improvement, you will need to rebuild your apps using MoSync SDK 3.2.1.
