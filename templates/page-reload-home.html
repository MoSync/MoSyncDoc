How to find your way around the Reload documentation

MoSync Reload is built on top of the MoSync SDK. Please note that the JavaScript Guides and JavaScript API References found in the SDK Documentation largely apply also to apps developed with Reload.

The parts of the SDK Documentation that do not apply to Reload are the guides and the example apps that use a hybrid application model, which mixes JavaScript and C++ code. The HTML/JS part of hybrid apps are however perfectly possible to test and develop using Reload. :)

Reload compared to the MoSync SDK

To find out about the differences between Reload and the MoSync SDK, examine the Reload/SDK Compared Guide.

Good to know - Upgrading projects

If you have projects created with an old version of Reload, make sure to update them. Specifically, you need to update the file wormhole.js to the latest version. Please read the Wormhole Upgrade Guide for further details.

Note that since Reload is built on top of the MoSync SDK, it is important to use the correct SDK version when you wish to build a native app from a Reload project. The following table shows the recommended versions:
